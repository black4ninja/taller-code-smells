Refactor Race

Taller de identificación y refactorización de code smells

TC3004B · ITESM

Duración: 1h 40min. Usa → o espacio para avanzar.

El objetivo de hoy

Implementar 7 funciones desde cero.

Pero con una trampa pedagógica: los tests no solo validan que funcionen. También miden:

  • líneas por función (LOC)
  • complejidad ciclomática
  • anidamiento
  • número de parámetros
  • uso de switch, if, += sobre strings
  • duplicación

Tu primer instinto al codificar hoy va a sufrir. Y a aprender.

¿Qué es un code smell?

«Un code smell es una pista superficial que habitualmente corresponde a un problema más profundo en el sistema.» — Martin Fowler, Refactoring (1999)

Clave: el código funciona, pero huele mal.

Smell = síntoma. Refactor = tratamiento. No los confundas.

Los buenos linters detectan algunos automáticamente. Para el resto: experiencia y revisión.

Los 11 code smells del temario

1
Rigidez
Cambiar X obliga a cambiar Y, Z, W.
2
Fragilidad
Un cambio rompe cosas no relacionadas.
3
Inmovilidad
No puedes reusar un componente fuera.
4
Viscosidad
El atajo malo es más fácil que el bueno.
5
Código duplicado
El mismo bloque en varios lugares.
6
Método largo
Más de 15-20 líneas. Hace muchas cosas.
7
Clase grande
God object. Demasiadas responsabilidades.
8
Lista larga de parámetros
Más de 3-4 args sueltos del mismo dominio.
9
Obsesión primitiva
Strings o ints para representar dominio.
10
Envidia de características
Método que usa más datos de otra clase.
11
Switch excesivo
switch(tipo) con muchas ramas.

Hoy tocaremos los más representativos en los 7 retos del taller.

Smell #11: Switch excesivo

Reto C1. Cada vez que agregas un tipo, tocas la función. Viola OCP.

antes 
function descuento(tipo, monto) {
  switch (tipo) {
    case 'regular': return 0;
    case 'plata':   return monto*0.05;
    case 'oro':     return monto*0.10;
    case 'diamante':return monto*0.15;
    case 'empleado':return monto*0.25;
    default: throw new Error('?');
  }
}
después 
const TASAS = {
  regular: 0, plata: 0.05, oro: 0.10,
  diamante: 0.15, empleado: 0.25
};

function descuento(tipo, monto) {
  const t = TASAS[tipo];
  if (t === undefined) throw new Error('?');
  return monto * t;
}

Técnica: Replace Conditional with Polymorphism (en JS: mapa de estrategias).

Smell #8: Lista larga de parámetros

Reto C2. Acompañado de anidamiento profundo (Rigidez).

antes 
function puedeAcceder(
  userId, role, banned,
  recursoId, ownerId,
  esPublico, ahora
) {
  if (role === 'admin') {
    if (!banned) {
      return true;
    } else { return false; }
  } else {
    // ...
  }
}
después 
function puedeAcceder(user, recurso) {
  if (user.banned) return false;
  if (user.role === 'admin') return true;
  if (recurso.esPublico) return true;
  return user.id === recurso.ownerId;
}

Técnicas: Introduce Parameter Object + Guard Clauses (early return).

Smell #9: Obsesión primitiva

Reto C3. Strings sueltos para algo que es un concepto del dominio.

antes 
function formatear(
  calle, numero, colonia,
  ciudad, estado, cp, pais
) {
  let r = '';
  if (calle) r += calle;
  if (numero) r += ' ' + numero;
  if (colonia) r += ', ' + colonia;
  // ... 4 ifs más con +=
  return r;
}
después 
function formatearDireccion(dir) {
  return [
    [dir.calle, dir.numero]
      .filter(Boolean).join(' '),
    dir.colonia,
    [dir.ciudad, dir.estado, dir.cp]
      .filter(Boolean).join(' '),
    dir.pais
  ].filter(Boolean).join(', ');
}

Técnicas: Parameter Object + filter(Boolean).join() en lugar de +=.

Smell #6: Método largo

Retos C0 y C4. Una función que hace 5 cosas tiene 5 razones para cambiar.

antes 
function total(items) {
  let s = 0;
  for (const it of items) {
    if (it.esEnvio) {
      s += it.precio * it.cantidad;
    } else {
      const sub = it.precio * it.cantidad;
      const iva = sub * 0.16;
      s += sub + iva;
    }
  }
  return s;
}
después 
const IVA = 0.16;

function precioConIva(it) {
  const base = it.precio * it.cantidad;
  return it.esEnvio
    ? base : base * (1 + IVA);
}

function total(items) {
  return items
    .reduce((s, it) =>
      s + precioConIva(it), 0);
}

Técnica: Extract Method. Una función = una idea.

El arsenal: técnicas + principios

9 técnicas de refactor

  • Extract Method — saca bloques cohesivos
  • Replace Conditional with Polymorphism — mata el switch
  • Introduce Parameter Object — agrupa args sueltos
  • Extract Class — conceptos del dominio
  • Move Method — ponlo donde pertenece
  • Compose Method — orquestador delgado
  • Guard Clauses — early return
  • Null Object — mata los if x === null
  • Dependency Injection — recibe, no instancies

Principios de fondo

  • SOLID
    • SRP: una razón para cambiar
    • OCP: abierto a extender, cerrado a modificar
    • LSP: los subtipos no rompen al padre
    • ISP: interfaces mínimas
    • DIP: depende de abstracciones
  • DRY — no te repitas
  • KISS — no compliques sin necesidad
  • Tell, Don't Ask — pídele al objeto que actúe
  • Law of Demeter — habla solo con tus amigos

Anatomía de un reto

  1. Abre tests/refactor/CN_*.test.js — lee qué pide
  2. Abre src/challenges/cN_*.js — aquí escribes
  3. Escribe la primera versión: lo primero que se te ocurra
  4. Corre npm run test:cN — verás rojo
  5. El mensaje del test te nombra el smell y sugiere la técnica:
  La función "calcularDescuento" tiene 24 líneas (límite: 10).
  Smell: Método largo.
  Técnica sugerida: Extract Method.
  Referencia: docs/REFACTOR_CHEATSHEET.md#extract-method
  1. Refactoriza. Vuelve a correr. Verde.
  2. Llena tu fila en docs/ENTREGABLE.md.

Reglas del race

  • Individual. Código tuyo, en tu máquina.
  • 1h 40min contados desde el banderazo.
  • C0 es guiado — los primeros 10 minutos lo hago yo en pantalla con docs/WALKTHROUGH.md.
  • Solo editas archivos en src/challenges/.
  • NO toques tests/refactor/ ni src/lib/quality/ — hay test de integridad que detecta trampa.
  • Gana quien tenga más retos en verde. Cuando termines, pulsa "Correr toda la suite" en el dashboard para validar.
  • Entregable paralelo: docs/ENTREGABLE.md con smell, técnica y reflexión por reto.

Tip: si te atoras más de 10 minutos en un reto, abre REFACTOR_CHEATSHEET.md y busca tu smell.
